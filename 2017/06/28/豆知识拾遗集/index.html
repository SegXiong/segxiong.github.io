豆知识拾遗集

2017.06.28 Seg Xiong Tips

最近的学习中学到的一些小知识点，由于不能单独成文，所以用此文做一次小结。

  • Rails 在 form_for 表单中，会默认命名 input 的 id 是 model 名称和該属性名称，这里也就是 post_content (此外还有默认的 name post[content] )。因此 jquery 可以用 $("#post_content") 抓到那一个 input 元素。

  • 在 chrome 的 inspect 中，选择 Network tab，再点击需要查看的 response 就是能看到 server 返回的 JSON 数据了。

  • form_for 会绑定一个 model 来做编辑，而 form_tag 是完全自定义的表单。所以我们可以自定义我们要的 input 例如 start_on。 所以在向后端传送的数据 params 中，不一定都是model中存在的column， 可能只是一个传递数据的自定义 input。 另外查看源码的话，会发现 < input name=start_on > name中并没有相应的 model 做前缀。

  • 在移除文件时 carrierwave 有一个内置的功能，会自动增加一个属性是 :remove_xxxx，如果这个属性有值，在更新 commit 的时候就会删除它。

  • 在同一个 routes 文件中， root “controller#action” 可以出现多次，前提是在不同的 namespace 下。
