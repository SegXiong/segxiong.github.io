Turbolinks

2017.06.09 Seg Xiong errors

在之前 Nic 助教的直播教程中，提到有很多同学反应 js 失效的问题都是因为 RoR 自带的 Turbolinks 的原因导致。

今天无意中在carts/checkout页面按到了刷新，结果页面出现如下的报错提示。

根据报错信息的提示，是controller中缺少show的action，而在RESTful中show是get方法。

问题是carts/checkout页面在代码中使用的是post方法（如图）。

所以查看了一下server log，果不其然日志中显示，第一次进入carts/checkout页面是使用的post方法，但是第二次（刷新）的时候却是get方法。

对此百思不得其解，最后slack的助教提示可能是turbolinks的原因，所以尝试删除掉turbolinks重走一遍出错流程，果然报错消失。

PS：这个错误只在chrome上才会出现，safari上即使刷新也是可以正确使用post方法的。猜测是chrome和turbolinks的冲突造成。

本来按着教程进行到这里，刚刚的问题也就解决了。但是由于魔改大赛，自己增加了很多新的代码上去，去掉turbolinks之后又出现了新的问题，flash不能在正确的时间显示。

在助教提示下研究server log发现了异常的地方如图

正常情况下在用所有server返回的都应该是html页面，只有在使用ajax之后才会返回js。在这个show页面的form中我确实使用了ajax

但是问题在于，删除turbolinks之前，使用ajax后页面也是正常的。

在一番Google之后，最后查看这篇文章找到原因。

所以最后的解决方案，也就有两种。一种是按那篇作者的方法，为页面添加js操作。另一种就是删掉remote： true这段代码，不使用ajax。

后记: 最终的解决办法是把 checkout 页面的路由 从 post 改为 get
